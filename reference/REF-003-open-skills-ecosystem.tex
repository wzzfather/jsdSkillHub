\section{概述}

\textbf{skills.sh} 是由 Vercel Labs 发起的开放 Agent Skills 生态系统。其核心组件包括：

\begin{enumerate}[nosep]
  \item \textbf{Skills CLI}（\texttt{npx skills}）--- 技能包管理命令行工具，MIT 许可
  \item \textbf{skills.sh} --- 技能浏览与搜索网站（类似 npm registry for agent skills）
  \item \textbf{agentskills.io} --- Agent Skills 开放规范
\end{enumerate}

npm 包名 \texttt{skills}，当前版本 v1.5.5，唯一依赖 \texttt{yaml}。维护者：rauchg（Vercel CEO）和 quuu。

该生态支持 \textbf{56+ 种 AI 编码 Agent}，包括 OpenClaw、Cursor、Claude Code、Codex、GitHub Copilot、Cline、Augment 等。

\section{Skill 格式规范}

\subsection{SKILL.md 结构}

每个 Skill 是一个目录，核心文件为 \texttt{SKILL.md}，格式为 YAML frontmatter + Markdown：

\begin{verbatim}
---
name: my-skill
description: What this skill does and when to use it
metadata:
  internal: true       # 可选，标记为内部技能
---

# My Skill

Instructions for the agent...

## When to Use
...

## Steps
1. First, do this
2. Then, do that
\end{verbatim}

\begin{table}[h]
\centering
\begin{tabularx}{\linewidth}{llX}
\toprule
\textbf{字段} & \textbf{必填} & \textbf{说明} \\
\midrule
\texttt{name} & 是 & 唯一标识，小写+连字符 \\
\texttt{description} & 是 & 功能描述，用于搜索发现 \\
\texttt{metadata.internal} & 否 & 设为 true 则默认隐藏，仅内部可见 \\
\bottomrule
\end{tabularx}
\end{table}

\subsection{Skill 发现位置}

CLI 在仓库中按以下顺序搜索 Skill：

\begin{enumerate}[nosep]
  \item 仓库根目录（如含 \texttt{SKILL.md}）
  \item \texttt{skills/}
  \item \texttt{skills/.curated/}（精选）
  \item \texttt{skills/.experimental/}（实验性）
  \item \texttt{skills/.system/}（系统级）
  \item 各 Agent 专属目录（如 \texttt{.claude/skills/}、\texttt{.cursor/skills/}）
\end{enumerate}

\subsection{插件 Manifest 兼容}

支持 Claude Code 插件市场的 manifest 格式：

\begin{verbatim}
// .claude-plugin/marketplace.json
{
  "metadata": { "pluginRoot": "./plugins" },
  "plugins": [{
    "name": "my-plugin",
    "source": "my-plugin",
    "skills": ["./skills/review", "./skills/test"]
  }]
}
\end{verbatim}

\subsection{CLI 命令清单}

\begin{table}[h]
\centering
\begin{tabularx}{\linewidth}{lX}
\toprule
\textbf{命令} & \textbf{功能} \\
\midrule
\texttt{npx skills find [query]} & 按关键词搜索技能 \\
\texttt{npx skills add <source>} & 安装技能（支持多种来源格式） \\
\texttt{npx skills list} & 列出已安装技能 \\
\texttt{npx skills update [skills]} & 更新已安装技能 \\
\texttt{npx skills remove [skills]} & 卸载技能 \\
\texttt{npx skills init [name]} & 创建新 Skill 模板 \\
\bottomrule
\end{tabularx}
\end{table}

关键选项：
\begin{itemize}[nosep]
  \item \texttt{-a, --agent <agents...>}：指定目标 Agent
  \item \texttt{-s, --skill <skills...>}：安装指定 Skill（\texttt{'*'} 表示全部）
  \item \texttt{-y, --yes}：跳过确认提示
  \item \texttt{--copy}：使用复制而非符号链接
  \item \texttt{--all}：安装到所有 Agent
\end{itemize}

\begin{table}[h]
\centering
\begin{tabularx}{\linewidth}{lll}
\toprule
\textbf{Agent} & \textbf{Project Path} & \textbf{Global Path} \\
\midrule
OpenClaw & \texttt{skills/} & \texttt{\textasciitilde/.openclaw/skills/} \\
Cursor & \texttt{.agents/skills/} & \texttt{\textasciitilde/.cursor/skills/} \\
Claude Code & \texttt{.claude/skills/} & \texttt{\textasciitilde/.claude/skills/} \\
Codex & \texttt{.agents/skills/} & \texttt{\textasciitilde/.codex/skills/} \\
GitHub Copilot & \texttt{.agents/skills/} & \texttt{\textasciitilde/.copilot/skills/} \\

\textbf{注意}：OpenClaw 的路径与其他 Agent 不同。Project 路径是 \texttt{skills/}（无点号前缀），Global 路径是 \texttt{\textasciitilde/.openclaw/skills/}。

\subsection{跨 Agent 兼容性}

Skills 遵循 [agentskills.io](https://agentskills.io) 开放规范，大部分功能跨 Agent 兼容。差异点：

\begin{table}[h]

\begin{tabularx}{\linewidth}{lX}

\textbf{特性} & \textbf{兼容性} \\
\midrule

\texttt{allowed-tools} & 除 Qoder、Zencoder 外均支持 \\

\bottomrule

\end{table}

\section{搜索与分发}

\subsection{skills.sh 网站}

\textbf{skills.sh} 是技能浏览与搜索网站，功能类似 npm registry：

\begin{itemize}[nosep]
  \item 搜索框：按关键词搜索 Skill
  \item Skill 详情页：名称、描述、安装量、来源仓库
  \item 安装量统计：如 \texttt{vercel-react-best-practices} 有 9.2K 安装
  \item 分类浏览：Web Development / Testing / DevOps / Documentation / Code Quality / Design / Productivity
\end{itemize}

\subsection{CLI 搜索}

\texttt{npx skills find [query]} 基于 skills.sh 索引进行关键词搜索，返回匹配的 Skill 列表及安装命令。

\section{可借鉴的设计}

\begin{enumerate}[nosep]
  \item \textbf{SKILL.md 格式}：YAML frontmatter + Markdown 简洁规范，可作为我们 manifest 文件的参考格式
  \item \textbf{Symlink 安装策略}：多 Agent 共享同一份 Skill 文件，单一真相源，值得借鉴
  \item \textbf{skills.sh 体验}：浏览/搜索/安装量统计的 Web 体验可作为前端参考
  \item \textbf{插件 Manifest 兼容}：\texttt{.claude-plugin/marketplace.json} 格式可考虑兼容
  \item \textbf{分层发现}：curated / experimental / system 三层分类机制
  \item \textbf{CLI 工具}：提供 \texttt{npx skills add} 级别的命令行安装体验
\end{enumerate}

\section{结论}

skills.sh 是一个轻量、开放的 Agent Skills 生态，核心优势在于\textbf{格式简洁}（SKILL.md）和\textbf{多 Agent 兼容}（56+）。但它面向开发者个人，缺少企业级必需的安全扫描、审批流程、权限控制和审计能力。

我们的企业应用商店在安全、审批、权限方面远超 skills.sh，但在\textbf{分发便利性}和\textbf{跨 Agent 兼容性}方面可以向其学习。建议后续迭代中：

\begin{itemize}[nosep]
  \item 考虑兼容 SKILL.md 格式，方便直接从 skills.sh 生态引入技能
  \item 参考 skills.sh 的浏览体验，优化我们的应用市场前端
  \item 提供类似 \texttt{npx skills add} 的 CLI 安装工具
\end{itemize}
