# 企业级应用商店 - 项目规则

## 项目背景
企业级 AI Agent 应用商店，支持内部技能和客户技能的分发、安全管控和生态运营。

## 核心需求（三大维度）
1. **产品功能**：应用注册/上传、语义搜索/发现、安装部署/配置、版本管理/回滚
2. **安全治理**：权限声明/最小化、三层安全扫描（静态+恶意软件+LLM语义）、审批工作流（条件路由）、运行时监控、审计追溯
3. **生态运营**：开发者体验/工具链、质量治理/评分、反馈闭环、社区建设/激励

## 技术选型范围
请从前端、后端、数据库、搜索引擎、安全扫描、消息队列、部署运维等维度，给出 2-3 个候选方案，并给出推荐方案和理由。

### 必须覆盖的技术选型
1. **前端框架**：Web 管理后台 + 开发者门户
2. **后端框架**：API 服务 + 审批工作流引擎
3. **数据库**：关系型（元数据）+ 文档型（扫描结果 JSON）
4. **搜索引擎**：向量语义搜索 + 全文检索
5. **安全扫描**：静态分析引擎 + 恶意软件扫描集成 + LLM 语义分析
6. **消息队列**：异步任务（扫描、通知）
7. **对象存储**：应用包存储
8. **认证授权**：OAuth 2.0 + RBAC
9. **容器化部署**：Docker + K8s
10. **监控告警**：运行时行为监控 + 异常检测

### 选型原则
- 优先开源方案，降低成本
- 考虑团队技术栈（假设为中小型团队 5-10 人）
- 企业级要求：高可用、可审计、可扩展
- 私有化部署能力
- 中文生态友好度

## 输出格式
Markdown 文档，包含：
- 每个维度的候选方案对比表
- 推荐方案及理由
- 技术架构总览图（用 Mermaid 语法）
- 风险评估与应对策略
