\section{需求背景}

D-011 MVP 阶段后端三层扫描已跑通，但前端仍为演示级别。需要按照 PRD 和 MASTER 总需求重新开发前端，达到正式应用商店的交互体验。

\section{当前问题}

\begin{enumerate}[nosep]
  \item 浏览页只有纯文本卡片，缺少搜索栏、分类标签、筛选排序
  \item 没有 Skill 详情页（PRD F-004）
  \item 没有用户认证页面（登录/注册，PRD F-001 前端部分已有但需优化）
  \item 没有"我的应用"页（开发者查看自己提交的 Skill 状态）
  \item 流程看板只是步骤说明，不是真正的管理界面
  \item 审批工作台功能正确但视觉粗糙，需优化
  \item 上传页缺少 ClamAV 展示和扫描进度反馈
\end{enumerate}

后端 API 保持不变，仅重新开发前端。色系：黑白灰简约。

\subsection{导航结构}

\begin{itemize}[nosep]
  \item 顶部导航栏：左侧品牌名"Skill Store"，右侧导航按钮
  \item 未登录：浏览、登录、注册
  \item 已登录：浏览、提交应用、我的应用、审批工作台（仅管理员）、退出
  \item 导航按钮当前页高亮
\end{itemize}

\subsection{交互细节}

\textbf{应用市场}：
\begin{itemize}[nosep]
  \item 默认显示全部，点击分类标签筛选
  \item 搜索栏输入关键词实时筛选（前端过滤即可，MVP 阶段不接 OpenSearch）
  \item 卡片点击跳转详情页
  \item 底部分页（每页 12 个）
\end{itemize}

\textbf{审批工作台}：
\begin{itemize}[nosep]
  \item 需要管理员权限，非管理员显示提示
  \item 表格列：Skill 名称、版本、分类、扫描摘要（Semgrep/ClamAV/LLM 各显示通过或警告图标）、提交时间、操作
  \item 点击行展开右侧抽屉（Drawer），显示三层扫描结果详情
  \item 每层扫描结果：通过/不通过标签 + 告警详情（JSON 格式化展示）
  \item 底部：通过按钮、驳回按钮、备注输入框
\end{itemize}

\textbf{提交应用}：
\begin{itemize}[nosep]
  \item 需要登录，未登录跳转登录页
  \item 两步：先填表单，再上传 zip
  \item 上传后自动轮询扫描状态，展示三层扫描进度
  \item 扫描完成后提示"已进入审批队列"
\end{itemize}
