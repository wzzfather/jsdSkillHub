# 需求变更记录

> 每次需求变更后，由 PM（OpenClaw）在 `demands/` 目录下生成对应的技术文档。
> 文档格式：Markdown（源） + PDF（交付物），使用 Tectonic 生成。

---

## 文档命名规范

```
demands/
├── DEMAND-001-role-redefine.md      ← 需求编号-简述.md（Markdown 源文件）
├── DEMAND-001-role-redefine.pdf      ← 对应的 PDF 交付物
├── DEMAND-002-xxx.md
├── DEMAND-002-xxx.pdf
└── ...
```

- 编号从 001 开始递增
- 简述用英文短横线连接（kebab-case）
- Markdown 是源文件，PDF 是交付物，两者同步提交

---

## 需求变更文档模板

每份文档包含以下章节：

```markdown
# DEMAND-XXX: 需求标题

**提出日期**：YYYY-MM-DD
**提出者**：魏子政
**优先级**：P0/P1/P2/P3
**状态**：待评估 / 已评估 / 开发中 / 已完成
**影响范围**：列出受影响的模块/文档

## 1. 需求描述
（原始需求，魏子政的原话或整理后的描述）

## 2. 背景与动机
（为什么需要这个需求）

## 3. 详细方案
（技术方案、实现思路）

## 4. 影响评估
### 4.1 技术影响
### 4.2 文档影响
### 4.3 进度影响

## 5. 任务拆解
| 任务 | 负责人 | 验收标准 |
|------|--------|---------|

## 6. 验收标准
（独立于 Cursor 自验收的 PM 验收标准）

## 7. 变更记录
| 日期 | 变更内容 |
|------|---------|
```
