# 演示 Skill 设计精髓（供界面展示）

> 用途：给前端页面提供一份可直接读取的演示文案，突出每个 Skill 的价值与边界。  
> 版本：v1.0.0  
> 维护：后端

---

## 1) 周报助手
- **版本**：`v1.0.0`
- **分类**：`productivity`
- **一句话定位**：自动把「日历安排 + 任务进展」整理成周报草稿，降低重复汇报成本。
- **核心输入**：
  - 时间范围（本周）
  - 日历事件列表（会议、评审、里程碑）
  - 任务记录（状态、负责人、阻塞项）
- **核心输出**：
  - 结构化周报草稿（完成项 / 进行中 / 风险与下周计划）
- **设计精髓**：
  - 不是替代思考，而是先生成“80 分初稿”，让人只做审核和补充。
  - 强调事实归纳（事件和任务）而非主观修饰，便于团队复用模板。

## 2) SQL 审计 Skill
- **版本**：`v0.9.1`
- **分类**：`security`
- **一句话定位**：对 `SELECT` 语句进行静态风险提示，提前暴露查询层面的潜在问题。
- **核心输入**：
  - SQL 文本（当前仅演示 `SELECT`）
- **核心输出**：
  - 风险清单（例如全表扫描风险、条件缺失、结果集过大提示）
- **设计精髓**：
  - 只做“提示与预警”，不直接改写 SQL，保证可控与可解释。
  - 通过轻量规则帮助开发者形成安全查询习惯，适合作为发布前门槛检查。

## 3) 工单摘要
- **版本**：`v2.3.0`
- **分类**：`support`
- **一句话定位**：把长文本工单压缩成极简摘要，提升客服与研发交接效率。
- **核心输入**：
  - 工单原文（问题描述、用户反馈、上下文日志）
- **核心输出**：
  - 三段式短摘要（问题 / 影响 / 建议动作）
- **设计精髓**：
  - 优先“可执行信息”而非完整复述，减少跨团队沟通噪音。
  - 统一摘要结构，方便后续自动分派和看板聚合。

## 4) 内部文档检索
- **版本**：`v1.4.2`
- **分类**：`knowledge`
- **一句话定位**：在受限知识库内做语义检索，快速返回可用内部信息。
- **核心输入**：
  - 检索问题
  - 指定知识库范围（部门、项目或文档集合）
- **核心输出**：
  - 命中文档片段 + 简短答案（可附来源标识）
- **设计精髓**：
  - “限定范围”优先于“全局搜索”，先保证答案可信和可追溯。
  - 面向企业场景，强调权限边界与知识隔离，降低信息泄露风险。
