\section{产品概述}

基于 MRD 分析的市场需求，本产品为企业内部提供 AI Agent 的统一管理平台。四大核心模块：应用市场、审批管理、安全扫描、用户权限。

\subsection{关键交互流程}

\textbf{应用提交流程}：开发者填写信息 $\to$ 上传 Agent 包 $\to$ 系统触发安全扫描 $\to$ 扫描通过 $\to$ 进入待审批队列 $\to$ 审批人审核 $\to$ 通过则上架，驳回则退回。

\textbf{应用安装流程}：用户浏览市场 $\to$ 查看详情 $\to$ 点击安装 $\to$ 系统记录安装关系 $\to$ 安装成功提示。

\section{约束与规则}

\subsection{工具链锁定}

仅使用 OpenClaw（PM AI）+ Cursor（开发 AI），Claude/Codex 暂不参与。

\subsection{拍板机制}

有歧义的环节，OpenClaw 必须先让魏子政拍板决定方向，再组织下一步的开发/验收/commit。禁止 OpenClaw 在未拍板的情况下自行假设。

\subsection{文档一致性维护规则}

当 PRD 与 MASTER 总需求文档内容不一致时：
\begin{enumerate}[nosep]
  \item 接口文本、功能描述、技术细节等局部不一致：以 MASTER 为主
  \item 涉及重大流程或业务流转路径：必须由魏子政拍板决定
\end{enumerate}

\section{验收标准}

\begin{itemize}
  \item[$\square$] 所有 P0 功能开发完成并通过测试
  \item[$\square$] 四种角色（管理员/审批人/开发者/普通用户）权限隔离正确
  \item[$\square$] 应用从提交到上架全流程可走通
  \item[$\square$] 搜索功能返回正确结果
  \item[$\square$] 前端页面在主流浏览器上显示正常
\end{itemize}
