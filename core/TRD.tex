\section{API 设计}

\subsection{接口规范}

\begin{itemize}[nosep]
  \item RESTful 风格，统一前缀 \texttt{/api/}
  \item 请求/响应格式：JSON
  \item 认证方式：Bearer Token（JWT）
  \item 分页参数：\texttt{page} + \texttt{page\_size}，默认 20 条/页
  \item 列表响应格式：\texttt{\{items: [...], total: N, page: N, page\_size: N\}}
  \item 错误响应格式：\texttt{\{detail: "错误描述", code: "ERROR\_CODE"\}}
\end{itemize}

\section{部署架构}

\begin{itemize}[nosep]
  \item Docker Compose 单机编排，开发/测试/生产统一
  \item PostgreSQL 数据卷持久化
  \item MinIO 数据卷持久化
  \item OpenSearch 数据卷持久化
  \item 后端通过 \texttt{uvicorn} 启动，生产环境加 \texttt{gunicorn}
  \item 前端生产构建后由后端静态文件服务或 Nginx 托管
\end{itemize}

\section{工具链与约束}

\begin{itemize}[nosep]
  \item \textbf{开发工具链}：OpenClaw（PM AI）+ Cursor（开发 AI），Claude/Codex 暂不参与
  \item \textbf{拍板机制}：有歧义时必须由魏子政拍板决定方向
\end{itemize}

\section{文档一致性维护规则}

当 TRD 与 MASTER 总需求文档内容不一致时：
\begin{enumerate}[nosep]
  \item 接口定义、数据库设计、路由设计等局部不一致：以 MASTER 为主
  \item 涉及重大流程或业务流转路径：必须由魏子政拍板决定
  \item 新增接口或修改接口需同步更新 PRD 功能清单
\end{enumerate}
