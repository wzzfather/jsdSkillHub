\section{架构方案}

\subsection{前端架构}

SPA 单页应用，分为管理后台（admin）和开发者门户（portal）两个区域。路由守卫做权限控制。API 调用统一封装 axios 实例，Pinia 管理共享状态。

\subsection{后端架构}

六层分层：routers（路由层）$\to$ services（业务层）$\to$ models（数据层）$\to$ schemas（模型层）$\to$ tasks（任务层）$\to$ utils（工具层）。全部 async/await。

\subsection{安全扫描架构}

三层扫描管道：上传 $\to$ Semgrep（静态代码）$\to$ ClamAV（恶意文件）$\to$ LLM 语义分析（权限合理性判断）$\to$ 汇总报告驱动审批。

\section{风险点}

\begin{itemize}[nosep]
  \item \textbf{OpenSearch 资源占用}：单节点模式下内存占用较高（建议 512MB-1GB）
  \item \textbf{LLM 扫描延迟}：语义分析依赖外部模型，可能成为审批流程瓶颈
  \item \textbf{MinIO 兼容性}：部分 S3 高级特性 MinIO 可能不完全支持
  \item \textbf{Vue 3 生态}：相比 React，部分 UI 组件库选择较少
\end{itemize}

\section{文档一致性维护规则}

当 RFC 与 MASTER 总需求文档内容不一致时：
\begin{enumerate}[nosep]

  \item 涉及重大流程或业务流转路径：必须由魏子政拍板决定
  \item 讨论点状态变更（如待讨论$\to$已决定）：需 MASTER 或魏子政拍板确认后更新
\end{enumerate}
